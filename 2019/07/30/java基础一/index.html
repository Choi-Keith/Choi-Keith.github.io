1.String、StringBuffer和StringBuilder有什么区别

  • String和StringBuffer、StringBuilder的区别在于String声明的是不可变的对象，每次操作都会生成新的String对象，然后将指针指向新的String对象，例如：

  • Jvm解析：首先创建对象s指向字符常量池”jvm”，然后创建一个新的对象s指向”jvm890”，而原来的对象则会变成垃圾被GC回收。

  • 而StringBuffer, StringBuilder可以在原有的对象基础上操作，每当我们用它对字符串做操作时，实际上是在同一个对象上操作的，所以在经常改变字符串内容的情况下不要使用String。

StringBuffer: 线程安全，但执行效率低一点，一般在多线程使用。
StringBuilder: 非线程安全，但执行效率高，一般用在多线程。

2.==和equals的区别是什么

==对于基本类型和引用类型==的作用是不相同的。
基本类型：比较的是值是否相同。
引用类型：比较的是”是否为同一个对象”，包括地址和数值都要相同才相同。

equals一般比较的时引用字符串对象里包含的字符序列是否完全相同。然而Object提供的equals()只是比较对象的数组，本质上Object类上的equals方法比较引用对象的结果与==运算比较引用对象的结果是完全一致的。只是String重写了Object的equals方法，把引用比较改成了值(只是单纯的字符序列，没有包括内存地址）比较。
提示: String x = “string”和String x = new String(“string”)是不一样的。String x = “string”的方式，Java虚拟机会将其分配到常量池当中，而String x = new String(“string”)则会被分配到堆内存中

3.hashCode和equals

为什么要有hashCode

  • 以HashSet为例，当把对象加入HashSet时，HashSet会先计算对象的hashCode值来判断对象加入的位置，同时也会与其他已经加入的对象的hashCode值作比较，如果没有相同的hashCode, HashSet就会假设对象没有重复出现(因为hashCode不相同，一定不是同一个对象, 但是hashCode相同的不一定是同一个对象, 还需要用equals比较才能确定)。但是如果发现有相同的hashCode值的对象，这时会调用equals()方法来检查hashCode相等的对象是否会真的相同。如果两者相同，HashSet就不会让其加入操作。

两个对象的hashCode()相同，则equals()也一定为true，对吗？

  • 答案是错误的，两个对象的hashCode()相同，equals()不一定为true。例如：
    String str1 = "通话";
String str2 = "重地";
System.out.println("str1:%d|str2:%d", str1.hashCode(), str2.hashCode());
System.out.println(str1.equals(str2));
    上面运行的结果是：
    str1: 1179395 | str2:1179395
    false
