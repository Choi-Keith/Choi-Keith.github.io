MySQL语法知识总结

发表于 2019-07-24 | 分类于 数据库

1.mysql开始使用

1.1 mysql的登录命令如下

mysql -h localhost -r root -p
提示 mysql为登录命令，-h后面是服务器的主机地址， 如果客户端和服务器在同一台机器上，host可以是localhost或IP地址，-u后面跟登录数据库的用户名称， 在这里是root，-p后面是登录密码

1.2 mysql已存在的数据库：

information_scheme: 存储元数据库，数据库对象信息，如用户信息，列信息，字符，分区等
performance_sheme: 存储数据库服务性能参数
mysql: 存储数据库用户权限信息
sys: 能够快速获取谁用了多少资源，有谁连接过

1.3 存储引擎分类

MyISAM: MyISAM拥有较高的插入、查询速度，但不支持事务，常用在web、数据仓储和其他应用环境下最常使用的存储引擎之一，在mysql5.5之前的版本中作为默认存储引擎。
INNODB: InnerDB是事务型数据库的首选引擎，支持事务安全表，支持行锁定和外键，是mysql5.5之后的默认存储引擎。
MEMORY: 内存型，访问速度非常快，但是不安全。

2.有关数据库的一些概念

3.逻辑模型的结构分类

3.1层次模型：层次模型是用属性结构来表示各类实体以及实体之间的联系。
3.2网状模型: 网状模型是用图形结构来表示各类实体以及实体之间的联系。
3.3关系模型: 关系模型是用二维表结构来表示各类实体以及实体之间的联系。
3.4面向对象模型：面向对象模型是用面向对象的观点来描述现实世界实体(对象)的逻辑组织、对象间相知和联系的模型。

4.关系模型中的常见术语

关系：对应通常所说的一张表。
元组：表中的一行即为一个元组。
属性: 表中的一列即为一个属性。
主键：表中的某个属性组，它可以唯一确定一个元组。
域：列的取值范围成为域。
分量：元组中的一个属性值。
关系模式：对关系中的描述。

键的分类：

  • 超键：在一个关系中，能唯一标识元组的属性或属性集称为关系的超键。
  • 候选键：如果一个属性集能唯一标识元组，且又不含有多余的属性，那么这个键称为关系的候选键。
  • 主键：如果一个关系中有多个候选键，则选择其中的一个键为关系的主键。
  • 外键： 如果一个关系表R中包含另一个关系表S的主键所对应的属性组F，则称此属性组F为关系表R的外键，并称关系表S为参照建，关系表R是依赖关系。

5.mysql数据类型

5.1整数数据类型

数据类型 字节数 有符号数取值范围 无符号数取值范围
TINYINT 1个字节 -128~127 0~256
SMALLINT 2个字节 -32768~32767 0~65535
MEDIUMINT 3个字节 -8388608~8388607 0~16777215
INT 4个字节 -2147483648~214783647 0~4294967295
BIGINT 5个字节 -9223372036854775808~9223372036854775807 0~18446744073709551615

提示：mysql支持在类型后面的括号指定该类型的显示宽度，例如INT(M)进行设置，其中M指示最大显示宽度
注意：显示宽度与存储数据的大小无关。数值的个数如果小于指定的宽度是，显示会由空格填充；如果插入了大于显示宽度的值，只要该值不超过该类型整数的取值范围，数据依然可以插入，而且还准确无误。

5.2浮点数类型

类型名称 字节数
FLOAT 4个字节
DOUBLE 8个字节
DECIMAL(M, D),DEC M+2个字节

提示：(M, D)，M表示总共的位数，D表示小数的位数，默认M值为10， D值为0；它的最大取值范围不是固定的，而是由M来决定的。
注意：数据库一般用DECIMAL来存储数据，因为浮点数在进行加减法运算和比较的时候很容易出现精度问题

5.3字符串类型

类型名称 字符长度
CHAR 0~255个字符
VARCHAR 0~65535个字符
TINYTEXT (2^8 - 1)个字符
TEXT (2^16 - 1)个字符
MEDIUMTEXT (2^24 - 1)个字符
LONGTEXT (2^32 - 1)个字符
SET set最大可以由64个成员

提示：CHAR和VARCHAR的区别
char: char为固定长度，在定义时指定字符串长度，如果输入的字符数小于指定的字符数，则系统会自动在左边用空格补全，它的存储长度在定义时就已经固定下来了。
varchar: varchar的长度时可变的，在定义的范围内，输入多少个字符，它就有多少个字符。它的存储长度是在输入时固定下来的。

5.4日期与时间类型

类型名称 日期格式 字节数
YEAR yyyy 1个字节
TIME HH:mm:ss 3个字节
DATE yyyy-MM-dd 3个字节
DATAETIME yyyy-MM-dd HH:mm:ss 8个字节
TIMESTAMP yyyy-MM-dd HH:mm:ss 4个字节

注意：日期和时间既可以用字符串设置也可以用数值设置，当时即使它们输入的内容是一致的，它们代表的含义可能不同。

7.主键与外键

7.1主键约束：要求键列的数据唯一，并且不允许为空。主键分为单字段主键和多字段主键

  • 单字段主键格式

  • 多字段联合主键

7.2外键约束：外键约束的主要作用是保证数据引用的完整性，定义外键后，不允许删除在另一个表中具有想关联的行。
语法规则：
[CONSTRAINT <外键名>] FROEIGN KEY 字段名1 REFERENCES <主表名> 主键列1

注意：从表的外键必须关联主表的主键，并且关联字段的数据类型必须匹配。

8.三大范式

第一范式

  • 第一范式是第二范式和第三范式的基础，它包含下列指导原则
    (1) 数据组的每个属性只可以包含一个值。
    (2) 关系中的每个数组必须包含相同数量的值。
    (3) 关系中的每一个数组一定不能相同。
    简单的来说就是表中的所有属性都是不可再分解的原子值，那么这个表就属于第一范式原则。

第二范式

  • 第二范式要求数据库表中的每个实例或行必须可以被唯一区分。

第三范式

  • 第三范式要求数据不能存在传递，非主键字段不能相互依赖

总结：简单的来说就是第一范式就是原子性，要求字段不能再分；第二范式就是完全依赖，没有部分依赖；第三范式就是没有传递依赖

9.完整性

9.1实体完整性

  • 数据库的一行就是实体。实体完整性是指主键约束、唯一约束、自动增长序列、联合主键等，它的作用是标志每一行数据不重复。

9.2域完整性

  • 域完整性是以单元格为单位进行约束，针对当前单元格的数据，也称列完整性。域完整性的约束有数据类型上的约束、非空约束(not null)和默认值约束(default)等。

9.3参照约束

  • 参照约束是值表与表之间的对应关系，通常可以建立两表之间的主键，外键关系，或编写两表的触发器来实现。操作有引用关系的数据表格过程中，对其一表格做插入、更新、删除等操作是，系统都会将被修改表格与另一张表格进行参照，从而阻止一些不正确操作。

10.视图

10.1视图是一个虚拟表，其内容由查询产生，与真实的表一样，包含行和列，行和列的数据来自定义视图的查询所引用的表，并且在引用视图时动态生成。
10.2视图特性：不存储具体的数据，可以跟基本表一样，进行增删改查，一般用作查询。
